🔗 テーブル関係

users → user_settings: 1対多（1ユーザーが複数のプラットフォーム設定を持つ）
users → user_prompts: 1対多（1ユーザーが複数のプラットフォーム別プロンプトを持つ）
content_metadata → platform_content: 1対多（1つのコンテンツが複数のプラットフォーム向けコンテンツを持つ）
users → content_metadata: 1対多（1ユーザーが複数のコンテンツを作成）

🔄 プロンプト統合フロー

1
グローバルキャラクタープロンプト取得
user_settings.global_character_prompt から全媒体共通のキャラクター設定を取得
2
媒体別プロンプト取得
user_prompts から該当プラットフォームの固定プロンプトを取得（なければデフォルト使用）
3
一時プロンプト適用
リクエスト時に指定された一時プロンプト（DB保存なし）を統合
4
最終プロンプト生成
キャラクター設定 + 媒体別設定 + 一時プロンプトを統合して AI に送信
5
使用プロンプト保存
platform_content に生成時に使用した各プロンプトを履歴として保存

✨ 実装済み機能

  • キャラクター設定: グローバルキャラクタープロンプト（全媒体共通）
  • 媒体別プロンプト: プラットフォーム固有の生成ルール
  • 一時プロンプト: 生成時のみ適用される追加指示
  • 履歴機能: 使用されたプロンプトの確認・コピー
  • 使用統計: プロンプトの使用回数・最終使用日時
